🧾 Use Case Document – HobbyFind

1. 🎭 액터 정의 (Actor Definitions)

액터명	유형	설명	주요 행동
비회원 사용자 (Guest)	Primary	로그인하지 않은 일반 방문자	취미 탐색, 카테고리 선택, 로그인·회원가입 페이지 이동
회원 사용자 (Member)	Primary	로그인한 사용자	취미 탐색, 북마크 추가/삭제, 마이페이지 확인, 로그아웃
시스템 (System)	Supporting	HobbyFind 웹 애플리케이션	페이지 렌더링, 데이터 관리(북마크, 세션), 인증 처리

2. 🧩 상세 Use Case 시나리오 (Detailed Use Case Scenarios)

UC-01. 메인페이지 탐색 (홈)

항목	내용
액터	비회원, 회원
목적	전체 취미를 탐색하고 관심 카테고리를 선택
시작 조건	사용자가 /에 접속
기본 흐름	1. 시스템은 Hero 섹션과 전체 취미 카드 그리드를 표시2. 사용자는 상단 필터(운동형/지능형/예술형)를 클릭 가능3. 선택 시 해당 카테고리 페이지(/category/[type])로 이동
결과	사용자는 관심 취미를 확인하거나 카테고리별 탐색으로 이동


⸻

UC-02. 카테고리별 페이지 탐색

항목	내용
액터	비회원, 회원
목적	특정 카테고리(운동형, 지능형, 예술형)에 속한 취미 탐색
시작 조건	사용자가 /category/[type] 접속
기본 흐름	1. 시스템은 선택된 카테고리명과 소개문을 상단에 표시2. 해당 카테고리의 취미 카드 목록 렌더링3. 회원일 경우 각 카드에 북마크 버튼 활성화
결과	사용자는 선택된 카테고리 내 취미를 탐색 가능


⸻

UC-03. 로그인

항목	내용
액터	비회원
목적	사용자 계정으로 로그인
시작 조건	비회원이 /login 접근
기본 흐름	1. 사용자가 이메일/비밀번호 입력2. “로그인” 버튼 클릭3. 시스템이 인증 검증4. 성공 시 세션 생성 후 / 또는 /mypage 리다이렉트
대체 흐름	로그인 실패 시 → 에러 메시지 표시 (“아이디 또는 비밀번호가 올바르지 않습니다.”)
결과	로그인 성공 → 회원 상태로 변경 (Top Bar: 마이페이지/로그아웃 표시)


⸻

UC-04. 회원가입

항목	내용
액터	비회원
목적	신규 계정 등록
시작 조건	비회원이 /signup 접근
기본 흐름	1. 사용자 입력 (아이디, 비밀번호)2. 약관 동의 체크3. “회원가입” 클릭4. 시스템이 계정 생성 후 /login으로 이동
대체 흐름	필드 누락/중복 아이디 시 → 에러 메시지 표시
결과	회원 생성 완료, 로그인 페이지로 이동 유도


⸻

UC-05. 취미 북마크 추가/삭제

항목	내용
액터	회원
목적	관심 있는 취미를 북마크 목록에 추가하거나 해제
시작 조건	회원이 로그인 상태에서 취미 카드 표시 화면에 진입
기본 흐름	1. 회원이 취미 카드의 북마크 아이콘 클릭2. 시스템이 북마크 여부 확인 후 상태 토글 (추가 또는 제거)3. UI에 반영 (하트 색상 변경 등)
대체 흐름	비회원이 북마크 클릭 시 → “로그인 후 이용 가능합니다” 알림 표시
결과	북마크 리스트에 해당 취미가 반영됨


⸻

UC-06. 마이페이지 조회

항목	내용
액터	회원
목적	북마크한 취미 확인 및 카테고리별 통계 보기
시작 조건	로그인된 사용자가 /mypage 접근
기본 흐름	1. 시스템이 북마크 목록을 불러와 카드 형태로 표시2. 카테고리별 북마크 분포를 차트로 시각화3. 북마크 해제 시 즉시 반영
대체 흐름	비회원이 접근 시 /login으로 리다이렉트
결과	사용자는 저장한 취미를 확인하고 통계를 시각적으로 인식 가능


⸻

UC-07. 로그아웃

항목	내용
액터	회원
목적	세션 종료 및 비회원 상태로 복귀
시작 조건	로그인된 사용자가 상단 “로그아웃” 클릭
기본 흐름	1. 시스템이 세션 정보 삭제2. /로 리다이렉트3. Top Bar가 비회원 상태로 전환
결과	로그아웃 완료, 비회원 UI로 변경

3. 🔁 주요 단계 및 이벤트 흐름 (Main Steps and Flow of Events)

단계	사용자 행동	시스템 반응	결과
1	홈페이지 접속 (/)	Hero 섹션 + 전체 취미 카드 표시	비회원 탐색 시작
2	카테고리 선택	/category/[type] 렌더링	해당 카테고리 취미 표시
3	로그인 클릭	/login 페이지 이동	로그인 폼 노출
4	로그인 성공	세션 생성, /로 이동	Top Bar 업데이트
5	북마크 클릭	데이터 업데이트	북마크 추가/삭제 반영
6	마이페이지 접근	/mypage 렌더링	북마크 목록 및 차트 표시
7	로그아웃	세션 종료	비회원 상태 복귀


⸻

4. ⚙️ 대체 흐름 및 엣지 케이스 (Alternative Flows and Edge Cases)

상황	대체 흐름 / 처리 방식
로그인 실패	에러 메시지 표시 (Invalid credentials)
비회원이 북마크 클릭	“로그인 후 이용 가능합니다” Tooltip 표시
비회원이 마이페이지 접근	/login으로 자동 리다이렉트
데이터 로딩 실패	로딩 스피너 표시 또는 “데이터를 불러올 수 없습니다.” 문구
네트워크 오류	fallback UI (“연결 상태를 확인해주세요.”)
빈 북마크 상태	“저장된 취미가 없습니다.” 메시지 + 홈 이동 버튼 표시


⸻

5. 🧱 사전조건과 사후조건 (Preconditions and Postconditions)

구분	내용
사전조건 (Preconditions)	- 사용자는 인터넷 연결 가능 상태- 브라우저 쿠키/세션 저장 가능- 회원의 경우 유효한 계정 존재
사후조건 (Postconditions)	- 로그인 시 세션 유지- 북마크 상태가 DB 또는 로컬에 저장- 로그아웃 시 세션 삭제 및 UI 초기화


⸻

6. 📋 비즈니스 규칙 및 제약사항 (Business Rules and Constraints)

항목	설명
회원제 기능 제한	북마크 및 마이페이지는 로그인 사용자만 이용 가능
데이터 변경 불가	취미 목록은 고정 (3개 카테고리 × 6개 취미)
중복 북마크 불가	동일 취미를 중복 저장할 수 없음
로그인 방식 고정	아이디/비밀번호 기반만 허용 (소셜 로그인 제외)
데이터 확장 제한	카테고리 추가 또는 취미 변경 불가
UI 일관성 유지	모든 페이지 동일한 Top Bar 구성 유지


⸻

7. 🚨 예외 처리 절차 (Exception Handling Procedures)

예외 상황	시스템 처리	사용자 메시지
잘못된 로그인 정보	에러 코드 401 반환	“아이디 또는 비밀번호가 올바르지 않습니다.”
인증 만료 (세션 만료)	자동 로그아웃 처리	“세션이 만료되었습니다. 다시 로그인해주세요.”
네트워크 오류	로딩 실패 시 fallback	“네트워크 연결을 확인해주세요.”
서버 오류 (500)	에러 페이지 노출	“일시적인 오류가 발생했습니다.”

8. 🧭 UI 고려사항 (User Interface Considerations)

요소	설계 기준
Top Bar	상태에 따라 버튼 세트 변경 (비로그인: 로그인/회원가입 / 로그인: 마이페이지/로그아웃)
Hero Section	간결한 안내 문구 (“나에게 맞는 취미를 찾아보세요”)
취미 카드	이미지 + 제목 + 카테고리 태그 + 북마크 아이콘
로그인/회원가입 폼	오류 메시지 시각적으로 구분 (빨간색 텍스트, text-red-500)
마이페이지	북마크 카드 그리드 + 통계 차트(원형 그래프, 색상별 카테고리 구분)
반응형 대응	모바일(2열) / 데스크톱(4열) 그리드 구성

9. 💾 데이터 요구사항 및 데이터 흐름 (Data Requirements and Data Flow)

데이터 항목	설명	소스	저장 위치
취미 데이터	카테고리별 취미 목록	정적 JSON	/data/hobbies.json
회원 데이터	이메일, 비밀번호, 북마크 목록	Supabase Auth / DB	Supabase
북마크 데이터	회원별 북마크한 취미 배열	클라이언트 액션 (POST/DELETE)	Supabase
세션 데이터	로그인 상태 관리	Auth 토큰	localStorage / Supabase session

데이터 흐름 예시

회원 로그인 → Supabase 인증 성공 → 세션 저장 → 
취미 카드 클릭(북마크 추가) → Supabase DB 업데이트 → 
마이페이지에서 북마크 목록 불러오기 → StatsChart로 시각화

10. 🔒 보안 및 개인정보 고려사항 (Security and Privacy Considerations)

항목	내용
인증 방식	이메일·비밀번호 기반 (Supabase Auth)
암호화 처리	비밀번호 해시 저장 (bcrypt 또는 Supabase 내장 암호화)
HTTPS 통신	모든 요청 HTTPS로 암호화
세션 관리	JWT 기반 세션 / 만료 시 자동 로그아웃
개인정보 수집 최소화	이메일·비밀번호만 저장, 이름/전화번호 등 불필요 정보 미수집
쿠키 사용	로그인 세션 유지 목적 (타 서비스 트래킹 없음)
데이터 접근 제한	마이페이지 및 북마크 API는 인증된 사용자만 접근 가능
