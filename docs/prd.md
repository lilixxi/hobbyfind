🧾 Product Requirements Document (PRD) – HobbyFind

⸻

1. 📘 제품 상세 설명 (Detailed Product Description)

항목	내용
서비스명	HobbyFind
서비스 타입	웹서비스 (반응형, Next.js 기반)
접근 조건	비회원: 탐색만 가능회원: 북마크 및 마이페이지 기능 이용 가능
서비스 목적	사용자가 운동형 / 지능형 / 예술형 카테고리로 분류된 취미를 탐색하고, 관심 있는 취미를 북마크하여 자신의 취향을 관리할 수 있도록 돕는 서비스
핵심 목표	- 단순하고 감각적인 취미 탐색 경험 제공- 로그인 기반으로 관심 취미 저장 및 관리 기능 구현- 카테고리별 북마크 통계로 자기 선호 취향 시각화

요약:
HobbyFind는 사용자가 새로운 취미를 탐색하고 자신의 관심사에 따라 북마크를 저장할 수 있는 탐색 + 관리형 웹서비스입니다.
비회원은 자유롭게 탐색만 가능하고, 회원은 로그인 후 북마크 및 마이페이지 기능을 이용할 수 있습니다.

4. 👤 사용자 페르소나 및 시나리오 (User Persona and Scenarios)

구분	내용
이름	김소현 (26세, 직장인)
특징	평일엔 일, 주말엔 쉬는 ‘무취미족’. 새로운 취미를 찾고 싶지만 뭘 시작해야 할지 몰라 탐색 중
목표	관심 있는 취미를 쉽게 찾아 저장하고, 나중에 다시 확인하고 싶음

시나리오 (비회원)	1️⃣ 사이트 접속 → 2️⃣ 카테고리 필터로 “예술형” 선택 → 3️⃣ 취미 카드 탐색만 가능 (북마크 버튼 비활성화 상태)

5. 🧠 기술 스택 추천 (Technical Stack Recommendations)

구분	기술	역할
프레임워크	Next.js (v14 이상)	SSR/SSG 지원, 라우팅 및 상태관리
언어	TypeScript	정적 타이핑, 유지보수성 향상
스타일링	TailwindCSS	반응형 UI 및 Airbnb 스타일 구현
데이터베이스	Supabase (PostgreSQL 기반)	회원 정보 및 북마크 데이터 저장
인증 관리	Supabase Auth	이메일/비밀번호 기반 인증 처리
차트 시각화	Chart.js or Recharts	마이페이지 내 북마크 분포 시각화
배포 환경	Vercel	Next.js 친화적 빌드/배포 환경
상태관리	React Query or Zustand	북마크/회원 세션 관리

6. 🚫 제한사항 (Limitations)

항목	설명
취미 데이터 확장 불가	3개 카테고리 × 6개 취미로 고정
소셜 로그인 미지원	아이디/비밀번호 기반 로그인만 가능
검색 기능 비포함	텍스트 검색, 자동완성 등 제공하지 않음
추천 알고리즘 미적용	북마크 기반 추천 또는 개인화 기능 제외
데이터 수정 제한	취미 데이터는 정적 JSON 기준, 관리자 수정 불가
오프라인 모드 미지원	인터넷 연결 필요
